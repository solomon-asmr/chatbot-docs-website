Embedded Chatbot Widget

Part of: Client-Side Components

This is the front-end chat interface that appears on the owner's website for Website Visitors. It's loaded via the integration snippet provided in the dashboard.

Appearance & Interaction:

  • Typically appears as a chat icon or window.
  • Provides a text input box for questions.
  • May include a microphone button for voice input.
  • May include an upload option for images/screenshots.
  • Designed to be intuitive and unobtrusive (e.g., familiar chat bubble).

Functionality:

  • Collects visitor input (text, voice, image) and sends it to the backend server via API call (REST/WebSocket).
  • Displays the AI's answer received from the backend.
  • Runs entirely in the visitor's browser (client-side) for responsiveness.
  • Handles voice capture (e.g., using Web Speech API) and image uploads locally if features are enabled.
  • Allows anonymous interaction – no visitor login required (session tracked internally).
  • Can handle follow-up actions like language selection or feedback submission.
The widget acts as the direct interaction point for website visitors seeking information from the AI assistant.

Voice Query Handler

Part of: Server-Side Components

Handles spoken questions from visitors when voice support is enabled, allowing interaction via voice instead of typing.

Typical Workflow:

  1. 1. Capture Audio (Client-Side)
    The chat widget records audio input from the visitor using the browser's microphone.
  2. 2. Send Audio (Client-Side)
    The recorded audio clip is sent to the backend server.
  3. 3. Transcribe Audio (Server-Side)
    The server uses an external Speech-To-Text (STT) service (e.g., OpenAI Whisper, Google Cloud Speech) to convert the audio into text.
  4. 4. Process as Text
    The resulting text query enters the standard Query Processing & RAG Pipeline.
  5. 5. Deliver Text Answer
    Retrieval and answer generation proceed as normal. The final answer is delivered back to the widget as text.
Accessibility

Provides an alternative input method, making the chatbot more accessible for users who prefer or need to speak rather than type.

Client-Side Option

Transcription can potentially be done client-side using the browser's Web Speech API to reduce server load (sending only text).

Enhancement

Optionally, the client widget could use Text-To-Speech (TTS) in the browser to read the text answer aloud.
