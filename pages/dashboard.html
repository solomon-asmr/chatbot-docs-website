Website Owner Dashboard

Part of: Client-Side Components

A web-based interface where the Website Owner (the person managing a site) can sign up, log in, and configure their chatbot.

Key Functions:

Configuration

Provide the website URL and any additional content/files for training. Customize settings like assistant name, welcome message, and feature toggles (voice, multilingual, etc.).

Integration

Generates a simple JavaScript integration snippet to embed the chatbot into the owner's website HTML. No advanced coding needed.

Easy Setup
Maintenance

Update the knowledge base by re-crawling the site or adding new documents.

Analytics

View analytics and chat logs (daily summaries, common questions) to understand user interactions.

This dashboard provides the primary interface for website owners to manage their custom AI assistant.

Multilingual Support (Translation) Module

Part of: Server-Side Components

This module enables the assistant to handle queries and provide answers in the visitor's preferred language, even if it differs from the website's primary content language (when the feature is enabled).

Key Aspects:

  • Language Detection/Preference: Can automatically detect the query language or use a stored preference from the visitor's session (e.g., "es" for Spanish).
  • Query Translation: If the query language differs from the site content language (e.g., Spanish query on an English site), it calls a Translation API (e.g., Google Translate) to translate the query *into* the site's language (English).
  • Standard Retrieval: The translated query (now in the site's language) is used for the normal RAG Pipeline (semantic search and content retrieval) against the indexed content.
  • Answer Generation: The LLM generates an answer based on the retrieved content, typically in the site's primary language (English).
  • Response Translation: The LLM's answer is translated *back* into the visitor's original language (Spanish) using the Translation API before being sent to the chat widget.
Goal: To provide an accurate answer, sourced from the website's content, delivered in the visitor's native language, bridging language barriers seamlessly.

Alternative approach: Prompting the LLM to answer directly in the target language is possible, but using a dedicated translation service often yields higher accuracy for retrieving information from source content before translation.
