B. System Architecture

The platform is designed with a modular, client–server architecture that separates the user-facing interface from backend processing.

Client Side Components:

  1. The website owner's dashboard for configuring the chatbot.
  2. The chat widget embedded on the owner’s website for visitor interactions.

Server Side Components:

On the server side, a backend application handles all heavy processing – including website crawling, content indexing, query understanding, and response generation – and interacts with both a database and external AI services.

  • Database: Stores website content (and their vector embeddings), chatbot configuration, and logs of Q&A interactions.
  • External APIs: Integrates with services like OpenAI (for LLM and embeddings) and others for Speech-to-Text or OCR.
